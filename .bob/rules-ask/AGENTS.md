# Ask Mode Rules (Non-Obvious Only)

## Hidden Documentation Locations
- **Security docs**: `docs/guides/AUTHENTICATION_GUIDE.md` has OAuth setup details
- **Algorithm details**: `archive/completed_analysis/SIMILARITY_ALGORITHM_CHANGE.md` explains Fréchet vs Hausdorff
- **Performance history**: `archive/completed_analysis/PERFORMANCE_*.md` files track optimization work
- **Release notes**: `docs/releases/HISTORICAL_RELEASES.md` has version history

## Counterintuitive Code Organization
- **`src/` is NOT the only source**: `app/models/` contains dataclass definitions used across project
- **Cache files are encrypted**: Plain JSON reading won't work - must use `SecureCacheStorage`
- **Two auth modules**: `src/auth.py` (legacy) and `src/auth_secure.py` (current) - use secure version
- **Test cache separate**: Production cache in `cache/`, test cache also in `cache/` but different filename

## Configuration Quirks
- **Environment vars in YAML**: Config file uses `${VAR_NAME}` syntax that gets replaced at runtime
- **Dot notation required**: Must use `config.get('strava.client_id')` not dict access
- **Multiple config files**: `config/config.yaml` (main) and `config/config_quick.yaml` (testing)

## Documentation Context
- **bob_skills.md**: Comprehensive development guidelines (1481 lines) - check this first for workflow info
- **TECHNICAL_SPEC.md**: Complete technical documentation in `docs/`
- **Plans directory**: Version-specific implementation plans in `plans/v*.*/`
- **Archive directory**: Completed analysis and historical docs in `archive/`

## Non-Obvious Features
- **Parallel processing**: Route analysis supports `--parallel 1-8` flag (not documented in basic help)
- **Cache merging**: `--fetch` merges by default, NOT replaces (counterintuitive)
- **Date range behavior**: When both `--from-date` and `--to-date` specified, `--limit` is ignored
- **Security audit log**: Separate from main logs, in `logs/security_audit.log`

## Issue Management Context
- **Priority system**: P0 (critical) → P1 (current sprint) → P2 (next sprint) → P3 (backlog) → P4 (future)
- **ISSUE_PRIORITIES.md**: Auto-generated by `./scripts/update-issue-priorities.sh` - don't edit manually
- **Weekly maintenance**: Review open issues, close completed, update labels every Monday
- **Commit references**: Use `Fixes #123` in commits to auto-close issues on merge

## Documentation Structure (Non-Standard)
- **Version-specific docs**: `docs/releases/v*.*/` contains release notes and time tracking per version
- **Plans directory**: `plans/v*.*/` has implementation plans, NOT just ideas
- **Archive vs docs**: `docs/archive/` for completed analysis, `archive/` for old code/scripts
- **Guides are separate**: `docs/guides/` for user-facing, `docs/` for technical specs
